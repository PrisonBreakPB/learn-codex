# 学习路线

本目录按“先看运行全景，再跟踪一次真实 turn，最后拆开各个 Harness 边界”的顺序组织。

所有源码结论均以 OpenAI Codex `rust-v0.145.0`（提交 `25af12f7e61572b0bc18ddb1008be543b91519b0`）为准。

## 每章的固定结构

1. **问题**：这一章要消除什么困惑。
2. **架构**：用图说明对象和边界。
3. **源码定位**：记录上游路径、符号与版本。
4. **运行轨迹**：只呈现由源码、测试或实验支持的事件顺序。
5. **最小实验**：验证一个单一机制，并写清前提与限制。
6. **科研 Harness 启发**：分为已证实事实、设计推断和待验证问题。

## 标记约定

| 标记 | 含义 |
| --- | --- |
| 上游事实 | 可直接追溯到固定版本的源码、测试或官方文档 |
| 学习归纳 | 为建立阅读路径而做的解释性分组 |
| 设计推断 | 面向科研智能体 Harness 的可迁移思考，不等同于 Codex 事实 |
| 待验证 | 现有证据不足，暂不下结论 |

## 如何复核

章节中的源码路径均相对上游仓库根目录。先确认本地快照处于固定提交，再用路径和符号搜索即可：

```powershell
$codexUpstream = "<本地 Codex 源码快照目录>"
git -C $codexUpstream rev-parse HEAD
git -C $codexUpstream describe --tags --exact-match
rg -n "<符号名>" (Join-Path $codexUpstream "<上游相对路径>")
```
